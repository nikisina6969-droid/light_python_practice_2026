## 2026-06-30 — Этап 2. Сканирование
- **Результат:** добавлен рекурсивный обход папки (`scanner.scan_folder`) со сбором
  пути, размера, даты изменения и типа файла. Индекс сохраняется в SQLite
  (`scanner.save_index`): новые файлы добавляются, существующие обновляются,
  исчезнувшие помечаются как `missing`. Добавлен вывод списка (`report.py`)
  и фильтр по расширению/имени. `main.py` связывает всё вместе.
- **Проблемы:** сначала фильтр применялся при сканировании и ошибочно помечал
  отфильтрованные файлы как отсутствующие. Перенёс фильтр на уровень вывода —
  индекс теперь всегда отражает реальное состояние папки.
- **Следующий шаг:** подсчёт хэшей и поиск дубликатов (этап 3).

## 2026-07-01 — Этап 3. Дубликаты
- **Результат:** добавлен модуль `duplicates.py`: подсчёт SHA-256 содержимого
  файлов (`file_hash`), сохранение хэшей в SQLite с переиспользованием между
  запусками (`update_hashes` пересчитывает только новые/изменённые файлы) и
  поиск групп с одинаковым хэшем (`find_duplicates`). Вывод групп — в
  `report.print_duplicates`, вызов по флагу `--duplicates` в `main.py`.
- **Проблемы:** чтобы не пересчитывать хэши каждый раз, сохраняю `mtime` на
  момент хэширования и сравниваю его при следующем запуске.
- **Следующий шаг:** сравнение с резервной копией и история проверок (этап 4).

## 2026-07-01 — Этап 4. Резервная копия
- **Результат:** добавлен модуль `backup.py`: сравнение исходной папки с копией
  (`compare` — отсутствующие, изменённые и лишние файлы; изменения определяются
  по размеру, а при совпадении — по хэшу) и сохранение результатов в SQLite
  (`save_check`, таблицы `backup_checks`/`backup_diffs`). Отчёт и история
  запусков — в `report.py`, вызов по флагам `--backup`/`--history` в `main.py`.
  README доведён до финального вида. Проект завершён.
- **Проблемы:** существенных не было; сверка по хэшу переиспользует функцию
